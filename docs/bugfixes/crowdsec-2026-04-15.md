# CrowdSec Bugfixes — 2026-04-15

## Bug #1: Traefik logs read but not parsed (OPEN)

**Symptom:** `cscli metrics show acquisition` shows `Lines read: 1, Lines unparsed: 1`. No alerts or decisions generated from Traefik traffic.

**What works:**
- CrowdSec Engine starts and registers with CAPI ✅
- Collections installed (traefik, http-cve, appsec) ✅
- Log file is visible and readable inside container ✅
- Manual `cscli explain --type traefik` parses lines correctly ✅
- Traefik access logs are valid JSON ✅

**What doesn't work:**
- Live acquisition doesn't parse incoming log lines ❌
- Custom s00-raw parser to set `program=traefik` didn't help ❌

**Root cause (suspected):**
The `traefik-logs` parser v1.5 filters on `evt.Parsed.program startsWith 'traefik'`. This field is populated by syslog/journald but NOT by file-based acquisition. Known issue: https://github.com/crowdsecurity/hub/issues/1677

Attempts:
1. Added `program: traefik` to acquis.yaml labels → didn't set `evt.Parsed.program`
2. Created local s00-raw parser to set `program` + `message` → parser loaded but live parsing still failed
3. Removed custom parser, tested vanilla config → still failed

**Status:** Parked. Engine runs, community blocklist syncs. Parser issue needs deeper investigation or upstream fix. Will revisit with Phase 2 (Traefik Bouncer).

**Next steps:**
- Monitor https://github.com/crowdsecurity/hub/issues/1677 for upstream fix
- Try Docker-based acquisition (`source: docker`) instead of file-based
- Or wait for parser v1.6+ that supports file acquisition natively
